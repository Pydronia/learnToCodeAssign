Images and Directories

At the moment, our website looks a little bland... why don't we add a image to spice things up? To do this, we add a <img> tag to our HTML - however, that's not all we need. We can't just expect the browser to know exactly what image we're talking about. We need to add an attribute to our <img> tag. There are lots of different attributes that we can use for different tags, but at the moment we are only going to talk about the src attribute for the <img> tag. The src attribute tells the <img> tag what image to actually display. We write it like this:

<img src="path_to_image" >

Essentially, attributes simply give elements more information to use. Before we go any further, it would be a good time to talk about directories.

Directories tell the browser, or computer, where it can find a specific file. For example, if you were looking for your documents folder on Windows, it's likely to be in the directory C:\Users\username\Documents\. On UNIX based systems (Mac, Linux, etc) it's likely to be in the directory /Users/username/Documents/, which can also be written as ~/Documents/ (the ~ symbol means your user folder in UNIX environments). As you can see, there are some differences between the two. The most obvious is that Windows uses backslashes (\) and UNIX uses forward slashes (/). Another difference is that Windows denotes what disk drive the computer should look on with the letter and the colon at the beginning of the path (C:\). The CGSComputing web server runs on a Linux based system, so it uses the UNIX type directory. This is what you will use to point to images and other resources in your HTML.

Let's say, in your root directory (usually the folder /) you have a folder called "images". Inside that folder you have a picture, say for example, meme.png. We want to get that meme so it displays on our website. To do this, in our HTML file which will also be in the root directory, we'll put a <img> tag with a source attribute that points to the image. If we use a directory source that doesn't have a / in front of it, it's called a relative directory. This is because it points to a folder from the directory the HTML file is currently in - e.g. in our case, we would write <img src="images/meme.png" > or if the HTML file was in another folder called HTML, we would write <img src="../images/meme.png" >. The ../ means that you go back in a directory, like pressing the back button in your file browser. If, however, the src attribute does have a / in front of it, it's called an absolute directory. This means instead of pointing to a directory from the HTML file, it points to it from the root directory.
