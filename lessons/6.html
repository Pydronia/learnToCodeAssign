Validation

Websites all have to follow rules to make sure they work properly and look as they should on websites - otherwise you can get weird glitches and inconsistencies between different browsers. This is where validation comes into play. There are many different types and versions of HTMl, however in these examples, we will be talking about XHTML 1.0 Strict validation.

So far most of the code you have already seen is based on HTML 5, the newest version of HTML. XHTML is a version of HTML that has very strict rules about syntax. This is what you must have in every HTML file you want to comply with XHTML:

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
<head>
    <title>Conforming XHTML 1.0 Strict Template</title>
    <meta http-equiv="content-type" content="text/html; charset=UTF-8" />
</head>
<body>

</body>
</html>

Note that the <!DOCTYPE> tag is very different, as well as the <html> tag. There is also a new tag, the <meta> tag - don't worry to much about this, but all it does is give extra information to the browser about the web page. Try now changing your current webpage to conform to XHTML (Hint: Change the DOCTYPE, html, and add the meta tags to match the code above), and validate your webpage at https://validator.w3.org/, the official HTML validator. Bookmark this page - you will find it very useful in the future!

You'll find that if you try to validate your website, you'll come up with a few errors. Try and fix these - a big problem for most new developers is remembering to put a / at the end of self-closing tags, like the <img> tag. Another thing to note about the <img> tag is that you will need a alt attribute as well, to validate to XHTML. This stands for "alternative", and it is simply the text to display if your image src does not point to a valid image, resulting in a broken image. Your <img> tag should now look like:

<img src="images/meme.png" alt="Cannot find the meme :(" />

Revalidate, and your web page should now comply completely to XHTML 1.0 Strict! If not, try to fix whatever other errors the validator shows. (The table may not validate properly, but don't worry about that for now as it requires CSS to fix.) Validation is very important to every web developer - it makes sure your code will work with new browsers far into the future - even if it looks OK now. Start trying to make all of your websites validate - even if the code we show doesn't validate properly. Doing this is great practice, and makes sure 100% that your web page will look and work like you want it too.

Organisation

Organisation is important for any big web based project - even this website uses organisation! If you have all of your websites, images, stylesheets, and other content all in one folder or even worse, the root directory, it makes it very hard to keep track of all of your files. It's great practice to spit up your images, CSS (which we will learn about later) and HTML into different folders depending on what they're for. This is most useful when working with multiple pages, which we will learn about in the next lesson!
